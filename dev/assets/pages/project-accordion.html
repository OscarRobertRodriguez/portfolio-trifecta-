How it works

Building an accordion with CSS was a little bit strange to me at first. Using label and radio buttons felt odd since there was no form tag in the code.

Either way it works and is a neat trick so let's dive in and see what's going on under the hood.

						
							
							
					     <div class="rainbow-accordion">
							<ul>
								<!-- Item 1 -->
								<li>
								    <!-- use radio button and label for our css magic  ----> 
									<input id="appetizers" type="radio" name="menu"/>
									<label for="appetizers">appetizers</label>
									<div class="content">
										<h3>Check our tasty apps!</h3>
										<h4>Happy hour menu - half off!!</h4>
										<p>Chicken Wings(6) - 5.99$</p>
										<p>Calamari - 7.99$</p>
										<p>Garlic breaded pickles - 4.59$</p>
									</div>
								</li>
								<!-- Item 2 --> 
								<!-- list repeats with other list items ---->

Nothing too fancy going on in the html. A list is used because just make more semantic sense for the code. Flexbox was used on the `rainbow-accordion` to make our whole list responsive.

We set the content class is given a 2px height so that it does not expand accordion. Then I set an overflow:hidden so that the content does not spill out of the content div. This sets the inactive state of each content divs. 

								
									.rainbow-accordion .content {
									    overflow:hidden;
									    padding: 0 1em;
									    width:inherit;
									    height: 2px;
									    border: inherit;
									    background-color:#fff;
									    transition: all .5s ease-in-out;
									    border:none;
									    background: #cfd8dc;
									    font-family: sans-serif;
									}

We style the label and that will act as our state handler. Every time another label is clicked it will check the radio button associated with it and uncheck the other label. Since we don't need to see the radio button I set it to display none.

								
									.rainbow-accordion__label {
									margin-top: 2px;
									border:1px solid black;
									padding:10px;
									font-size:20px;
									color:#641C19;
									letter-spacing: 1px;
									font-weight: 500;
									font-family: sans-serif;
									background-color:#DB5A3F;
									text-transform:uppercase;
									transition: background-color .5s ease-out ;
									display: block; // this allows the label to fill in the li tag
									position: relative;
									cursor:pointer;
									transition: all 1s ease-in-out;
								}
								/* hides radio buttons */
								.rainbow-accordion [type="radio"] {
									display: none;
								}

Now this is the cool part on how we can leverage css to do something you might think would require JavaScript. Now I love JavaScript and for more complex components or a bigger site there is nothing better to use but in this case CSS works.

I use the the general sibling selector also known as the tilde ~ (til-duh). What this does is it will select every thing that comes after it. So if we had : 

                            	

                            		h2 ~ p {
										color: red;
									}
										/* will select every p tag if its below the h2 tag and a sibling */

All the paragraphs become red. We use the tilde. Now we use the tilde to add styling to the content div and the label when the radion button is checked.

                            	

                            		.rainbow-accordion [type="radio"]:checked ~ label {
									    color: #fff;
									    background-color:#C1263F;
									    font-weight:600;
									}
									// when radion is checked do this to content div
									.rainbow-accordion [type="radio"]:checked  ~ .content {
									    height: 200px;
									    border: 1px solid black;
									    border-top: none;
									}

When a label is clicked that will select the radio button right above the label. Then using the tilde we can say to expan the content and also add a different color to label.

It's a nifty use of css and definitely one that I will continue to use where it's appropriate.
